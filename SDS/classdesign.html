4.0 Class Design

4.1 Class Description

1.User: We are using user class to store all the data about a user. Everytime a user registers to our system a new row of data will be stored and also for sign in we will match data from this class.

2.Organizer Team: Users create a organizer team to organize a competition. It may contain one or more user. The organizer team will create competition and will give update about the competition.

3.Competition: Organizer team creates the competition and any user will be able to participate in the competition. It will store the data about the users who are going to participate and also about the organizer who are organizing this.

4.Query: Many participants may have different set of queries related with the competition. So, we are creating a forum where any one will be able to ask anything. Queries will be stored against the userId.

5.Update: The competitions may have many updates which the participants must know. So, the organizer team will be able to give any update post.

6.Category: Every competition must belong to a competition. The competition class will store the categoryId from which the competition belongs to.

4.3 Details of each class

User:

it will contain all types of users. Such as: Participants and organizers. The attributes of this class is: Id; password; name; competitionId(foreign key); organizerTeamId(foreign key); current institution.

Organizer Team:

this class is made up by one or multiple users. Any user can create a organizer team, add members. The attributes of this class is: Id; userId(foreign key); competitionId(foreign key); OrganizerName.

Competition:

Organizer team creates the competition and Participants can participate in the competition. There will also be a forum and discussion panel in every competition. The attributes of this class is: Id; userId(foreign key); organizerTeamId(foreign key); name; venue; date; deadline.

Query:

Every competition have a forum. Where any one can ask anything about the competition. And admin/any other user will answer to it. The attributes of this class is: Id; competitionId(foreign key); userId(foreign key); organizerTeamId(foreign key); comments.

Update:

Every competition will have a discussion page. Where the admin will post any update related with the Competition. Anyone will be able to comment on it. And admin will answer to it. The attributes of this class is: Id; competitionId(foreign key); userId(foreign key); organizerTeamId(foreign key); updateText; comments.

Category:

It is an abstract class. Every competition must belong to any category. The attributes of this class is: Id; categoryName.
